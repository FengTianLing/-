移动端默认样式
1.点击高亮效果 
-webkit-tap-highlight-color:transparent;  清除默认高亮

2.盒子模型:在移动端一般要保证盒子的大小不变
box-sizing:border-box;

3.input的样式:现在的手机input按钮都有扁平化效果,为了PC端一致,都会清除默认样式
甚至移动端有些浏览器会加上一些特有样式,比如3D高光效果
border:0;
-webkit-appearance:none;


4.最小和最大宽度:在移动端开发时,有时候会限制一些元素的最大值和最小值
max-width:680px;min-width:680px;

5:移动端的宽度问题:为了适配机型宽度,在宽度给值方面按百分比进行适配。高度由内容撑开

6.视网膜屏幕:分辨率和像素点是2倍或3倍的关系;精灵图使用都会进行倍数压缩
background-size:200px 200px;

7.移动屏幕适配布局方式
    宽度百分比,定高
    左边定宽,右边百分比自适应

移动端特有事件:
1.不能使用on的方式进行绑定,只能通过addEventListener
2.主要三个主要部分组成
    touchstart:触摸开始事件
    touchmove:触摸移动事件
    touchend:触摸结束事件
3.主要封装:左滑,右滑,长安,捏合。都是3个主要部分的封装

点击事件:一般会封装一个叫'tap'的方法 点击事件
手指点上去,不移动,快速松开;判断touchmove事件是否执行了(变量标识)

屏幕适配
1.1 rem适配
    em 是一个相对长度单位,参照的是当前元素的字号的px长度;假设盒子宽度为20em 字体是16px 那么盒子的宽度是20*16=320px 
    注意:浏览器默认字体大小是16px ,汉字大小最小不能小于12px. 但是字体大小用62.5% 那么大小就是10px  10/16 = 0.625
    rem 也是相对长度单位,参照的是html元素的字号的px长度;假设 同上;用途:当前元素字号改变,盒子大小不变的时候就用rem 这样rem的计算值就不会采用当前元素的字号来计算
